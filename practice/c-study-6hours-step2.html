1시간차: 문법을 구현으로 다시 묶기

여기서는 "안다"가 아니라 "바로 쓸 수 있다"를 목표로 합니다. 각 항목을 10분 내외로 짧게 돌리세요.

연습 1. 함수와 반환값

int add(int a, int b) {
    return a + b;
}
  • 함수 선언과 호출을 직접 3번 써본다.
  • return이 없으면 어떤 일이 생기는지 말로 설명해본다.
  • 입력 2개, 반환 1개 구조를 익힌다.

연습 2. 배열과 반복문

int arr[5] = {1, 2, 3, 4, 5};
for (int i = 0; i < 5; i++) {
    printf("%d\n", arr[i]);
}
  • i++, i += 2를 각각 써본다.
  • 배열 길이와 인덱스 차이를 말로 구분한다.
  • 오프바이원 실수를 직접 한 번 경험해보고 고쳐본다.

연습 3. 구조체

struct Student {
    int id;
    int score;
};
  • 구조체는 "관련된 값을 하나로 묶는 틀"이라고 설명할 수 있어야 한다.
  • s.id, s.score 값을 출력해본다.
  • 구조체 전체를 한 번에 출력하지 못하고 멤버별로 출력한다는 점을 기억한다.
체크 질문: 배열, 포인터, 구조체가 각각 무엇을 저장하는지 재혁님 말로 설명할 수 있으면 1시간차는 성공입니다.

2시간차: 포인터를 진짜로 이해하기

자료구조는 결국 "주소를 저장하고 따라가는 기술"입니다. 포인터를 넘기지 못하면 Linked List도 막힙니다.

핵심 개념

  • int x = 10;: 값 저장
  • int* p = &x;: x의 주소 저장
  • *p: 그 주소가 가리키는 실제 값 접근

필수 예제

#include <stdio.h>

int main(void) {
    int x = 10;
    int* p = &x;

    printf("x = %d\n", x);
    printf("&x = %p\n", (void*)&x);
    printf("p = %p\n", (void*)p);
    printf("*p = %d\n", *p);

    *p = 20;
    printf("x = %d\n", x);
    return 0;
}

여기서 꼭 느껴야 하는 것

  • p와 *p는 완전히 다르다.
  • p는 주소, *p는 실제 값이다.
  • 값을 직접 바꾸지 않아도 포인터를 통해 바꿀 수 있다.

실수 포인트

  • 초기화 안 된 포인터 사용
  • NULL 체크 없이 접근
  • 주소와 값 개념 섞기
목표: "주소를 따라가서 값을 바꾼다"는 말을 예제 없이도 설명할 수 있으면 다음 단계로 넘어가면 됩니다.

3시간차: Linked List를 직접 손으로 만들기

이 단계에서 재혁님은 "배열과 다르게, 연결 리스트는 다음 칸이 아니라 다음 노드의 주소로 이동한다"는 감각만 제대로 잡으면 됩니다.

노드 구조

struct Node {
    int data;
    struct Node* next;
};

가장 작은 예제

struct Node* head = NULL;
struct Node* second = NULL;
struct Node* third = NULL;

head = (struct Node*)malloc(sizeof(struct Node));
second = (struct Node*)malloc(sizeof(struct Node));
third = (struct Node*)malloc(sizeof(struct Node));

head->data = 10;
head->next = second;

second->data = 20;
second->next = third;

third->data = 30;
third->next = NULL;

이 시간의 연습 순서

  1. 3개 노드 연결하기
  2. while (current != NULL)로 출력하기
  3. 맨 앞 노드 1개 더 추가해보기
  4. 맨 끝에 노드 추가해보기

재혁님이 꼭 말할 수 있어야 하는 문장

  • Linked List는 각 노드가 다음 노드의 주소를 저장하는 구조다.
  • head는 첫 노드를 가리킨다.
  • current = current->next;는 다음 노드로 이동하는 코드다.
추천 연결 파일: 저장소의 Linked List 폴더 첫 문제를 열고, 이미 있는 코드 틀을 읽어보세요. "이 함수는 head를 어디서 받고 어디를 바꾸는가?"만 봐도 공부가 됩니다.

4시간차: Stack / Queue는 결국 규칙이 다른 저장 구조

자료구조를 처음 배울 때는 "새로운 마법"처럼 보이지만, 실제로는 넣는 규칙과 빼는 규칙이 다른 것뿐입니다.

Stack

나중에 들어간 것이 먼저 나온다. LIFO.

Queue

먼저 들어간 것이 먼저 나온다. FIFO.

배열 기반 Stack 예제 틀

int stack[100];
int top = -1;

void push(int x) {
    top++;
    stack[top] = x;
}

int pop(void) {
    int value = stack[top];
    top--;
    return value;
}

이 시간 목표

  • top, front, rear가 어떤 뜻인지 구분한다.
  • push/pop, enqueue/dequeue를 손으로 따라가 본다.
  • 배열 버전과 Linked List 버전 차이를 말로 설명해본다.

권장 미션

  • 숫자 3개를 push 후 pop 결과 예상하기
  • 숫자 3개를 enqueue 후 dequeue 결과 예상하기
  • 비어 있을 때 pop/dequeue 하면 어떻게 처리할지 직접 정해보기

5시간차: Tree와 재귀를 두려워하지 않기

Tree는 "노드 하나가 자식 포인터를 여러 개 가질 수 있다"로 이해하면 됩니다. 완벽한 재귀 구현보다 구조 읽기가 먼저입니다.

기본 구조 예시

struct TreeNode {
    int data;
    struct TreeNode* left;
    struct TreeNode* right;
};

핵심 관찰 포인트

  • Linked List의 next 하나가 Tree에서는 left, right 두 개로 늘어난 느낌이다.
  • 루트, 왼쪽 자식, 오른쪽 자식 개념을 그림으로 그릴 수 있어야 한다.
  • 순회는 "현재 노드에서 무엇을 하고 어느 방향으로 갈지"의 규칙이다.

재귀를 볼 때 질문할 것

  1. 이 함수는 언제 멈추는가?
  2. 현재 노드에서 무엇을 처리하는가?
  3. 왼쪽과 오른쪽 중 어디로 다시 호출하는가?
void preorder(struct TreeNode* node) {
    if (node == NULL) return;
    printf("%d\n", node->data);
    preorder(node->left);
    preorder(node->right);
}
재혁님 단계에서는 재귀를 완벽히 외우기보다, "멈추는 조건"과 "다음 호출 방향" 두 가지만 놓치지 않는 게 더 중요합니다.

6시간차: 현재 저장소 문제로 바로 연결하기

이제 연습용 작은 파일에서 벗어나 실제 문제 파일로 들어갑니다. 여기서 중요한 건 한 문제를 다 맞히는 것보다 코드 흐름을 읽고 한 칸씩 수정하는 힘입니다.

추천 진입 순서

  1. Linked List 첫 문제 파일 열기
  2. 입력/출력 형식보다 함수 역할부터 읽기
  3. 주어진 틀에서 비어 있는 부분만 채우기
  4. 작은 입력으로 직접 테스트
  5. 출력이 틀리면 print 디버깅으로 좁히기

문제 읽을 때 체크리스트

  • 입력은 몇 번 들어오는가?
  • 반복문은 언제까지 돌아야 하는가?
  • 배열로 할지, 포인터/구조체로 할지 어떤 구조가 필요한가?
  • 반환값이 필요한가, 아니면 포인터를 직접 바꾸는가?

막혔을 때 순서

  1. 컴파일 에러 먼저 1개씩 처리
  2. 출력값 틀리면 중간 값을 출력
  3. 포인터면 주소와 값 둘 다 확인
  4. 마지막 노드나 경계 조건을 다시 본다
